🎮 Cómo instalar Hekate + Ubuntu en Nintendo Switch:
Guía simplificada

2025-03-27 - Alba Tolosa Bonora

  • Requisitos
  • Instalación de Hekate
  • Modo RCM
  • Instalación de Ubuntu
  • Configuración
  • Tips
  • Créditos

Nota: Esta guía está basada en Ubuntu Jammy 22.04 + Switchroot. Asegúrate de seguir los pasos y siempre hacer doble-check.

Aquí encontrarás los enlaces a todos los recursos que necesitas para el proceso.
⚠️ Caution: No muevo tus manos ni soy responsable de tus Bricks.

Requisitos

  • Nintendo Switch compatible → Comprueba tu SN aquí
  • Tarjeta SD (mínimo 16GB, recomendado 64GB/128GB U3/A2)
  • Hekate 6.0.6+ instalado
  • PC para preparar la SD y los archivos
  • Conexión a internet, paciencia y mucha suerte... 🙃
    Jejej va... no tanta.

⚙️ Instalación de Hekate

Si es la primera vez que instalas Hekate, sigue estos pasos:

    1. Descarga la última versión de Hekate desde su página de GitHub.
    2. Descarga la última versión de TegraRcmGUI desde su página de GitHub. Necesario para inyectar los payloads y encender la Switch.
    3. Obligatorio sólo si instalas CFW. Si no, puedes pasar:
      • Enciende la consola como siempre.
      • Accede a los ajustes de internet y borra todas las redes wifi recordadas. (Evita descuidos = baneos)
    4. Apagar completamente la consola.
    5. Extrae la SD y conéctala al PC.
    6. Extrae el contenido del ZIP de Hekate en la raíz de la SD.
    7. Pon la Switch en modo RCM, conéctala al pc e inyecta el Payload de Hekate desde TegraRCM.
    8. Curiosea Hekate.

⚡ ¿Cómo iniciar Hekate desde el modo RCM?

Para poder arrancar Linux o cualquier otro sistema alternativo como Android, necesitas entrar en modo RCM y lanzar el bootloader Hekate desde tu PC o teléfono.

Pasos para entrar en modo RCM y cargar Hekate:

  1. Prepara tu PC o móvil con el payload sender
    Un payload actúa como la "llave" que te permite acceder a la puerta de entrada del sistema. Necesitarás un programa para enviar el archivo .bin de Hekate a tu consola:
    • Windows: El más conocido → TegraRcmGUI
    • Linux/macOS: fusee-nano (Linux) o CrystalRCM (Mac)
    • Android: apps como NX Loader
      Inyecta el payload sin necesidad de abrir la app (requiere cable OTG)
  2. Conecta tu Switch al PC por USB-C
  3. Pon la Switch en modo RCM
    • Apaga la Switch completamente
      (mantén el botón de encendido y selecciona Apagar)
    • Utiliza un Jig para simular la pulsación de los botones requeridos.

    Pasos:

    1. Inserta el jig en el rail derecho de los Joy-Con
    2. Mantén presionados los botones Volumen + y Power
    3. La pantalla se quedará en negro pero encendida (esto a veces puede causar confusión...), significa que está en modo RCM
  4. Lanza el payload de Hekate
    Con la Switch conectada y en modo RCM, abre el programa TegraRcmGUI
    (o el que uses) y:
    • Verás un mensaje como: RCM detected
    • Carga el payload hekate_ctcaer_(x.x.x).bin
    • Haz clic en "Inject payload"
    • La Switch arrancará en Hekate y tu serás feliz 😁

⭐ Desde Hekate puedes:

  • ✔️ Crear particiones en la SD
  • ✔️ Hacer Backups (eMMC / EmuMMC)
  • ✔️ Instalar → Ubuntu / Android / CFW
  • ✔️ Activar el modo RCM Auto (Para no tener que usar el Jig al encender)
  • ✔️ Elegir el sistema a utilizar → (Nintendo - Linux - Android - Lakka - CFW)
  • ✔️ Cargar otros payloads

Asegúrate de seguir cada paso cuidadosamente para evitar problemas durante la instalación.

🎮 Pasos de Instalación

1. Descargar imagen de Ubuntu Jammy

  • Desde la Switchroot Wiki
  • Usa torrent o descarga directa desde Aquí
  • Extrae el .7z a la raíz de la partición FAT32 de la SD (usa 7-Zip, no el descompresor de Windows 11)

2. Backup y particionado

⚠️ ¡Este paso borra todo el contenido de la SD!

  • Si ya tienes un CFW instalado con emuMMC en RAW, entra en Hekate y ve a: Tools → Backup eMMC. Luego, selecciona SD emuMMC RAW Partition y haz una copia de seguridad (emuMMC BOOT0/1 y emuMMC Raw GPP).
    ⚠️ Si no usas emuMMC, puedes saltarte este paso.
  • Luego: Tools → Seleccionar Partition SD Card dejando al menos 6 GB para el HOS en FAT32

3. Flashear Linux

  1. Tras realizar la partición conecta la switch al PC, usa UMS (USB Tools)
  2. Copia el contenido de la carpeta ubuntu-jammy a la raíz SD
  3. Desconecta de forma segura desde el PC
  4. Selecciona Flash Linux

4. Dump de Joy-Cons

  • En Hekate: Nyx Options → Dump Joy-Con BT
  • Esto es obligatorio incluso con Switch Lite (guarda calibración)

5. Primer arranque de Ubuntu

  • Desde la pantalla inicial de Hekate ve a More Configs → Ubuntu Jammy
  • El sistema se cargará desde la SD
  • ¡Ya estás dentro!

🛠️ Configuración

  • En el primer arranque puede que necesites un teclado físico (USB conectado al Dock o mediante adaptador OTG) para completar la configuración inicial de Ubuntu. Esto sólo si falla el Onboard / Accesibilidad.
  • Una vez dentro, abre el menú de aplicaciones y ejecuta el LT-4 Megascript que estará esperandote para instalar utilidades y controladores esenciales (incluyendo soporte mejorado para JoyCons, teclado virtual, etc.).
  • Si no salta automáticamente la accesibilidad, activa el teclado en pantalla desde Configuración → Accesibilidad → Teclado en pantalla o buscándolo como “Onboard”. Aunque tras ejecutar el script debería mostrarse si no lo hacía antes.
  • Instala VS Code y otras apps útiles a través de Flatpak (ya viene configurado por defecto en Ubuntu Jammy para Switch).
  • Ajusta la resolución de pantalla o configura gestos táctiles desde la configuración de GNOME.

Es bueno saber que...

  • Después del primer arranque y ejecución del Megascript, los JoyCons pueden funcionar como ratón/teclado (según el nivel de configuración aplicado).
  • Puedes conectar la Switch a un monitor externo usando el Dock original o un adaptador USB-C a HDMI, y usarla como mini PC de escritorio.
  • Los periféricos 2.4GHz como teclados o ratones con dongle USB funcionan perfectamente cuando se conectan al Dock o mediante un hub USB.
  • Si el teclado virtual se cierra solo, puede deberse a drift en los JoyCons. Intenta calibrarlos o usar otros Joy-Con.
  • Debido a que la Switch tiene una arquitectura ARM64 (no x86 como la mayoría de PCs), puede haber incompatibilidades o limitaciones con ciertas aplicaciones o servicios. Yo tuve problemas intentando levantar MongoDB, ya que no hay soporte oficial para ARM en algunas versiones.
